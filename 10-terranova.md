# 10. Terranova étape par étape

Ce guide pas-à-pas accompagne l’activation des modules Terranova pour un membre Semisto. Il reprend les éléments d’onboarding détaillés dans Compass, mais condensés ici.

## Étape 0 – Pré-requis
- Compte Google Semisto + appartenance au groupe `terranova-beta` (ou `terranova-prod` quand la v1 sera lancée).
- Navigateur Chrome/Brave recommandé.
- Avoir lu le chapitre Outils du Starter Kit (section Terranova).

## Étape 1 – Connexion
1. Ouvre `https://terranova.semisto.org`.
2. Clique “Sign in with Google”.
3. Vérifie ton pôle principal (Design / Academy / Nursery / Roots) → cela conditionne les modules visibles.

## Étape 2 – Tableau de bord
- Sur la home, repère les widgets
  - Pipeline projets Design
  - Sessions Academy à venir
  - Stocks Nursery (lecture seule pour l’instant)
  - Événements Roots
- Clique sur “Guided tour” si proposé (mode bêta).

## Étape 3 – Gestion des contacts / organisations
1. `CRM → Contacts` : crée un contact type (client, bénévole, partenaire) avec tags.
2. Ajoute une note ou un document (devis, photo) → vérifie que le fichier est stocké.
3. Associe le contact à un projet, une session Academy ou un événement Roots.

## Étape 4 – Projets Design
1. `Design Studio → Projets`
2. Crée un projet test “Forêt-jardin DEMO”
3. Renseigne les phases OBREDIM (Observation → Maintenance)
4. Ajoute un plan (PDF) et un calendrier d’interventions
5. Mentionne l’équipe projet + sponsor client

## Étape 5 – Sessions Academy
1. `Academy → Sessions`
2. Planner un atelier (date, lieu, capacité)
3. Ajouter des formateurs + supports
4. Inscrire des participants (import CSV ou fiche contact)
5. Générer la feuille de présence et les emails (templates disponibles)

## Étape 6 – Roots / événements
1. `Roots → Événements`
2. Créer une journée Semisto Day ou un chantier participatif
3. Définir les rôles bénévoles recherchés
4. Publier la page (si le module site est actif) ou exporter la liste pour newsletter

## Étape 7 – Nursery (aperçu)
- Visualise les stocks existants (grille par espèces, nombre, statut)
- Ajoute une fiche espèce (description, climat, période de plantation)
- Note les actions (semis, bouture, greffe) dans la timeline

## Étape 8 – Suivi & reporting
- Utilise les rapports prêts à l’emploi (revenus, formations, bénévoles)
- Exporte les données si nécessaire (CSV).
- Documente tout bug / friction dans Slack `#terranova-beta` + ticket Notion/Trello (selon workflow Bee).

## Étape 9 – Bonnes pratiques
- Encode les activités le jour même (CRM, projets, événements).
- Ajoute des tags explicites (#beta-feedback, #priority) pour le produit.
- Utilise les notes Terranova pour documenter une décision plutôt que des DM Slack.

## Étape 10 – Aller plus loin
- Suivre les formations Terranova (drive + Loom) quand la v1 sera prête.
- Lire les release notes (section Terranova du Starter Kit).
- Participer aux Terranova Days (sessions Q&A avec Bee / Super Génial).

> 🛠️ Ce guide évoluera avec la mise en production de nouveaux modules (Nursery complète, finances, API publics). Pensez à proposer des améliorations via PR.
